# `/book epub` — render chapters to EPUB

Render Markdown chapters to an EPUB file suitable for upload to Amazon KDP (digital Kindle pipeline) or any e-reader. Symmetric with `/book pdf`.

## Usage

```
/book epub <book> [chNN]
```

- `<book>` — the book directory under `chapters/` (e.g. `book-1`, `book-2`, `book-3`).
- `chNN` — optional. If provided, packages only that chapter; otherwise packages the whole book (all `chNN.md` concatenated, with auto TOC).

## Output

- single chapter → `chapters/<book>/epub/chNN.epub`
- whole book → `chapters/<book>/epub/<book>.epub`

The `epub/` subdirectory is created automatically. The user's project should add `chapters/*/epub/` to its `.gitignore` (one-time project-side setup; not the skill's responsibility).

## Steps

1. Verify `chapters/<book>/` exists in the project working directory. If missing, stop and tell the user.
2. Run the build script (self-bootstraps Python deps via `uv` shebang — first invocation resolves them and caches the env; subsequent runs are instant):
   ```
   ~/.claude/skills/book/scripts/build_epub.py <project_root>/chapters/<book> [--chapter chNN]
   ```
3. The script prints `wrote <path>` on success — surface that path to the user.
4. If invocation fails with `command not found: uv` (or equivalent), surface the install hint and stop:
   ```
   curl -LsSf https://astral.sh/uv/install.sh | sh
   ```
   Do NOT attempt to install dependencies automatically.

## Metadata override (optional `meta.yaml`)

To set proper metadata for KDP upload, the user can drop a `meta.yaml` next to the chapters at `chapters/<book>/meta.yaml`:

```yaml
title: The Real Title of the Book
author: Author Name
identifier: urn:isbn:9781234567890   # or any stable URI
language: en
```

All keys are optional. Defaults:
- `title` → first `# ...` of `chapters/<book>/outline.md`, fallback to directory name
- `author` → `"Unknown Author"`
- `identifier` → deterministic `urn:uuid:...` derived from the book directory path (stable across re-renders, KDP-friendly)
- `language` → `"en"`

PyYAML is now a required dependency (declared in the script's PEP 723 header), so `meta.yaml` Just Works once `uv` is on the PATH.

## Notes

- Typography lives in `~/.claude/skills/book/scripts/epub.css` — reflow-friendly, em-based, no fixed page sizes (e-readers handle pagination). Edit that file to adjust look-and-feel.
- Chapter order in whole-book mode is `sorted(glob("ch*.md"))` — names like `ch01.md`, `ch02.md`, ... must be zero-padded for correct ordering.
- Cover images are NOT generated by this command. KDP accepts the cover at upload time; if you need the cover embedded in the EPUB itself, that's a follow-up feature.
- This command is on-demand: it is NOT part of the canonical writing pipeline.
